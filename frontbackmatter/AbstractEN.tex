\begin{otherlanguage}{american}
	\chapter*{Abstract}
This work investigated the language-specific performance overhead of OpenTelemetry in AWS 
Lambda for Java, Python, and Node.js.
Based on five microbenchmarks, instrumented and non-instrumented functions were compared in 
a controlled environment.
The performance metrics runtime, memory consumption, and initialization times were measured and 
statistically evaluated.

The results show that Python has the lowest overhead, 
followed by Java and Node.js. Node.js causes extremely high overheads, particularly in Init Duration.
In absolute performance, Python is the fastest language in nearly all scenarios.
A cost analysis confirms that Python is the most economical option in 4 out of 5 benchmarks, 
while Java is only more cost-effective for asynchronous Lambda invocation.

For practical applications, Python is recommended for most synchronous use cases, while Java remains 
competitive in asynchronous Lambda chaining scenarios.
