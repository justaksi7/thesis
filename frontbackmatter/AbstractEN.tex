	\chapter*{Abstract}
The motivation for this work is the increasing use of distributed tracing with OpenTelemetry in serverless architectures. While OpenTelemetry is very useful for monitoring distributed systems, it also causes a significant performance overhead in terms of runtime, memory, and cold start times. However, there has been no systematic investigation so far of how this overhead affects different programming languages.

This work investigates and empirically evaluates the language-specific overhead of OpenTelemetry in AWS Lambda for Java, Python, and Node.js.
Instrumented and non-instrumented functions are compared based on five microbenchmarks which represent the five most 
common use cases of a FaaS (Function as a Service).
The performance metrics runtime, memory consumption, and cold start times are measured in a controlled environment and statistically evaluated 
with regards to the five microbenchmarks.

The results of this work will be included here after completion of the experiments.
